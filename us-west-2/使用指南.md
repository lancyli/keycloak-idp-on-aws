# 使用指南 — Keycloak Realm + Amazon Quick / QuickSight SSO 联合配置

这是本 CDK 项目部署的高可用 Keycloak（Keycloak 26.x）的**部署后一次性配置**。CDK 只负责部署 Keycloak 本身；Realm、客户端、映射器以及 AWS 侧的 IAM SAML 提供商在这里配置。

涵盖两条联合（federation）路径：

- **Amazon Quick Desktop** → **OIDC** 公开客户端（PKCE）。
- **Amazon Quick / QuickSight Web 控制台** → **SAML 2.0** 客户端，联合进入 AWS 登录端点（`https://signin.aws.amazon.com/saml`）。

全文中请替换以下占位符：

| 占位符 | 含义 | 示例 |
|--------|------|------|
| `<KEYCLOAK_URL>` | `keycloak-ha-cloudfront.KeycloakPublicUrl` 输出 | `https://d111111abcdef8.cloudfront.net` |
| `<ACCOUNT_ID>` | 客户 AWS 账号 ID | `123456789012` |
| `<REALM>` | Realm 名称 | `aws-realm` |
| `<SAML_PROVIDER>` | IAM SAML 提供商名称 | `keycloak` |
| `<FED_ROLE>` | 用户联合进入的 IAM 角色 | `keycloak-quicksight-federation` |

> **AWS ZHY（ZHY）：** 所有 `signin.aws.amazon.com` 变为 `signin.amazonaws.cn`，
> 所有 `arn:aws:` 变为 `arn:aws-cn:`，SAML 受众（audience）相应改变。
> 详见 `README.md` 中的"ZHY适配要点"。

## 0. 前置条件

获取部署生成的管理员密码和访问地址：

```bash
# 公网访问地址
aws cloudformation describe-stacks --stack-name keycloak-ha-cloudfront \
  --region us-west-2 --query "Stacks[0].Outputs[?OutputKey=='KeycloakPublicUrl'].OutputValue" \
  --output text

# 管理员密码（用户名为 'admin'）
aws secretsmanager get-secret-value --secret-id keycloak-ha/keycloak/admin \
  --region us-west-2 --query SecretString --output text
```

以 `admin` 身份登录 `<KEYCLOAK_URL>/admin/`。

## 1. 创建 Realm

**管理控制台** → 左上角 Realm 切换器 → **Create realm**。

- Realm 名称：`aws-realm`
- 点击 Create。

随后进入 **Realm settings**：

- **General** → 按需设置显示名称。
- **Login** → 启用**暴力破解检测**（Realm settings → Security defenses → Brute force detection → On）。
- **Sessions/Tokens** → 保持默认；SAML 断言有效期应 ≥ 下文设置的 AWS `SessionDuration`。

26.x 中 `sslRequired` 默认为 `external`（对非私有地址强制 HTTPS），与 CloudFront 的 HTTPS 前门一致，无需修改。

至少创建一个测试用户：**Users → Add user**，然后 **Credentials → Set password**（测试账号请关闭 *Temporary*）。

## 2. Amazon Quick Desktop —— OIDC 公开客户端

**Clients → Create client**。

- Client type：**OpenID Connect**
- Client ID：`amazon-quick-desktop`
- Next。
- **Client authentication：Off**（公开客户端）。
- **Standard flow：On**；Direct access grants：Off。
- Next。
- **Valid redirect URIs**（桌面客户端使用的回环地址 + 自定义 scheme）：
  - `http://localhost/*`
  - `http://127.0.0.1/*`
  - `quick://callback`
- **Valid post logout redirect URIs**：同上（或填 `+`）。
- Save。

**Advanced → Proof Key for Code Exchange (PKCE)**：设置 **Code challenge method = S256**。

桌面客户端使用的发现/端点：

```
Issuer:        <KEYCLOAK_URL>/realms/aws-realm
Discovery:     <KEYCLOAK_URL>/realms/aws-realm/.well-known/openid-configuration
Authorization: <KEYCLOAK_URL>/realms/aws-realm/protocol/openid-connect/auth
Token:         <KEYCLOAK_URL>/realms/aws-realm/protocol/openid-connect/token
```

验证：

```bash
curl -s <KEYCLOAK_URL>/realms/aws-realm/.well-known/openid-configuration | jq .issuer
# -> "<KEYCLOAK_URL>/realms/aws-realm"
```

## 3. Amazon Quick / QuickSight Web —— SAML 客户端

这一步让 Keycloak 成为 AWS 登录端点的 SAML IdP。

**Clients → Create client**。

- Client type：**SAML**
- Client ID：`urn:amazon:webservices`  *（这是 AWS 期望的 SAML `entityId`/受众）*
- Next。
- **Valid redirect URIs / Assertion Consumer Service POST Binding URL**：
  `https://signin.aws.amazon.com/saml`
  *（ZHY：`https://signin.amazonaws.cn/saml`）*
- Save。

**Client → Settings**：

- **Name ID format**：`persistent`（若以 email 作为 AWS 会话标识，则用 `email`）。
- **Force POST binding**：On。
- **Sign assertions**：On（Keycloak 默认用 realm 密钥签名）。
- **Client signature required**：Off（AWS 不对其 AuthnRequest 签名）。

### 3a. AWS 必需的 SAML 属性映射器

**Client → Client scopes →** `urn:amazon:webservices-dedicated` **→ Add mapper →
By configuration**，添加 AWS 要求的三个映射器：

**(i) Role（用户可担任的 IAM 角色）** —— *Role list* 映射器：

| 字段 | 值 |
|------|-----|
| Mapper type | Role list |
| Name | `aws-role` |
| Role attribute name | `https://aws.amazon.com/SAML/Attributes/Role` |
| Friendly name | `Role` |
| SAML attribute NameFormat | `Basic` |

属性**值**必须是 `<role_arn>,<provider_arn>` 这对组合：

```
arn:aws:iam::<ACCOUNT_ID>:role/<FED_ROLE>,arn:aws:iam::<ACCOUNT_ID>:saml-provider/<SAML_PROVIDER>
```

从 Keycloak 角色映射：创建一个 realm 角色（如 `quicksight-user`），把它的 **Role attribute** 描述设为上面这对值，再将用户分配到该角色。（若所有用户都担任同一角色，也可用 hardcoded-attribute 映射器。）

**(ii) RoleSessionName** —— *User Property* 映射器：

| 字段 | 值 |
|------|-----|
| Mapper type | User Property |
| Name | `aws-session-name` |
| Property | `username`（或 `email`） |
| SAML attribute name | `https://aws.amazon.com/SAML/Attributes/RoleSessionName` |
| NameFormat | `Basic` |

**(iii) SessionDuration** —— *Hardcoded attribute* 映射器：

| 字段 | 值 |
|------|-----|
| Mapper type | Hardcoded attribute |
| Name | `aws-session-duration` |
| SAML attribute name | `https://aws.amazon.com/SAML/Attributes/SessionDuration` |
| Attribute value | `3600`（秒；≤ 角色的最大会话时长） |
| NameFormat | `Basic` |

### 3b. 导出 IdP 元数据

下载 realm 的 SAML 描述符（创建 IAM 提供商时需要）：

```bash
curl -s <KEYCLOAK_URL>/realms/aws-realm/protocol/saml/descriptor -o keycloak-idp-metadata.xml
```

## 4. AWS 侧 —— IAM SAML 提供商 + 联合角色

### 4a. 创建 SAML 身份提供商

```bash
aws iam create-saml-provider \
  --name <SAML_PROVIDER> \
  --saml-metadata-document file://keycloak-idp-metadata.xml
# 记录返回的 SAMLProviderArn：
# arn:aws:iam::<ACCOUNT_ID>:saml-provider/<SAML_PROVIDER>
```

### 4b. 创建联合角色

`trust-policy.json` —— 针对 AWS 登录受众，信任该 SAML 提供商：

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "Federated": "arn:aws:iam::<ACCOUNT_ID>:saml-provider/<SAML_PROVIDER>"
      },
      "Action": "sts:AssumeRoleWithSAML",
      "Condition": {
        "StringEquals": {
          "SAML:aud": "https://signin.aws.amazon.com/saml"
        }
      }
    }
  ]
}
```

```bash
aws iam create-role --role-name <FED_ROLE> \
  --assume-role-policy-document file://trust-policy.json \
  --max-session-duration 3600
```

附加**最小权限**的 QuickSight 权限（生产环境务必收紧——不要直接用 `quicksight:*`）：

```bash
# 仅为示例——请收窄到你实际需要的 QuickSight 动作/资源。
aws iam put-role-policy --role-name <FED_ROLE> --policy-name quicksight-access \
  --policy-document '{
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Action": ["quicksight:CreateUser", "quicksight:RegisterUser"],
      "Resource": "*"
    }]
  }'
```

这里的 `arn:aws:iam::<ACCOUNT_ID>:role/<FED_ROLE>` **必须**与你在 Keycloak Role 属性（第 3a-i 步）中填写的角色 ARN 一致。

## 5. 测试 SAML 流程

1. **IdP 发起**：在浏览器打开
   `<KEYCLOAK_URL>/realms/aws-realm/protocol/saml/clients/urn:amazon:webservices`
   ，以测试用户登录 → 应以联合角色身份登入 AWS 控制台。
2. 若提示 *"Your request included an invalid SAML response"*，按顺序检查：Role 属性对的顺序是否为 `role,provider`；提供商元数据是否为最新；断言是否已签名；`SAML:aud` 是否与 ACS URL 匹配。
3. 对 QuickSight 而言，首次联合登录会预置 QuickSight 用户（受角色策略约束），随后将用户引导至 QuickSight 起始 URL。

## 6. 可选：自动化 Realm 配置（替代控制台点击）

为便于客户可重复交付，可导出已配置的 realm 再重新导入，或用一次性 Kubernetes Job 驱动 `kcadm.sh`：

```bash
# 在 keycloak pod 内部：
kubectl -n keycloak exec -it deploy/keycloak -- \
  /opt/keycloak/bin/kcadm.sh config credentials \
    --server http://localhost:8080 --realm master \
    --user admin --password "$KEYCLOAK_ADMIN_PASSWORD"

# 将完整配置的 realm 导出为可版本化、可重新导入的文件：
kubectl -n keycloak exec -it deploy/keycloak -- \
  /opt/keycloak/bin/kcadm.sh get realms/aws-realm/partial-export \
    -r aws-realm > aws-realm-export.json
```

在全新部署上用 `kcadm.sh create realms -f aws-realm-export.json`（或 `keycloak-config-cli`）重新导入。这样就把上述手动步骤变成一个随项目交付的单一制品。
