# 使用指南 — Keycloak Realm + SAML/OIDC 联合配置（AWS 中国 / ZHY）

本 CDK 项目在 **cn-northwest-1** 部署的高可用 Keycloak（Keycloak 26.x）的**部署后一次性配置**。CDK 负责部署 Keycloak 本身；Realm、客户端、映射器以及 AWS 侧的 IAM SAML 提供商在这里配置。

**这是 AWS 中国（`aws-cn`）版本。** 所有 AWS 登录端点为 `signin.amazonaws.cn`（**不是** `.com`），所有 ARN 为 `arn:aws-cn:...`。公网入口是 **ALB**（无 CloudFront）。

请替换以下占位符：

| 占位符 | 含义 | 示例 |
|--------|------|------|
| `<KC_URL>` | ALB 前置的访问地址（Stack 输出 `AlbDnsName`，或你的 ICP 域名） | `https://idp.example.cn` 或 `http://k8s-...elb.cn-northwest-1.amazonaws.com.cn` |
| `<CN_ACCOUNT_ID>` | 客户 AWS 中国账号 ID | `123456789012` |
| `<REALM>` | Realm 名称 | `aws-realm` |
| `<SAML_PROVIDER>` | IAM SAML 提供商名称 | `keycloak` |
| `<FED_ROLE>` | 用户联合进入的 IAM 角色 | `keycloak-quicksight-federation` |

---

## 0. 前置条件

```bash
# ALB DNS 名称（公网入口）
aws cloudformation describe-stacks --stack-name keycloak-ha-cn-eks \
  --region cn-northwest-1 --query "Stacks[0].Outputs[?OutputKey=='AlbDnsName'].OutputValue" \
  --output text

# 管理员密码（用户名为 'admin'）
aws secretsmanager get-secret-value --secret-id keycloak-ha-cn/keycloak/admin \
  --region cn-northwest-1 --query SecretString --output text
```

以 `admin` 身份登录 `<KC_URL>/admin/`。

> 若管理控制台出现加载 / "something went wrong" 错误，说明 `KC_HOSTNAME` 尚未固定。
> 在 `lib/config.ts` 中把 `keycloak.publicUrl` 设为 `<KC_URL>` 并重新部署
> （`npx cdk deploy keycloak-ha-cn-eks`）。参见 README 的 "HTTPS / KC_HOSTNAME（两阶段）"。

## 1. 创建 Realm

管理控制台 → Realm 切换器 → **Create realm** → 名称 `aws-realm` → Create。

- **Realm settings → Security defenses → Brute force detection**：On。
- `sslRequired` 默认为 `external`（对非私有地址强制 HTTPS）——当用 ACM 证书 / ICP 域名前置 ALB 时保持该设置。仅 HTTP 测试时可临时设为 `none`。
- 创建测试用户（**Users → Add user**，然后 **Credentials → Set password**，关闭 *Temporary*）。

---

## 2. Amazon Quick Desktop —— OIDC 公开客户端

**Clients → Create client**

- 类型：**OpenID Connect**，Client ID：`amazon-quick-desktop` → Next
- **Client authentication：Off**（公开），**Standard flow：On** → Next
- **Valid redirect URIs**：`http://localhost/*`、`http://127.0.0.1/*`、`quick://callback`
- Save → **Advanced → PKCE → Code challenge method = S256**

端点：

```
Issuer:    <KC_URL>/realms/aws-realm
Discovery: <KC_URL>/realms/aws-realm/.well-known/openid-configuration
```

---

## 3. Amazon Quick / QuickSight Web —— SAML 客户端

**Clients → Create client**

- 类型：**SAML**，Client ID：`urn:amazon:webservices`（AWS 期望的 SAML 受众）→ Next
- **ACS / Valid redirect URI**：`https://signin.amazonaws.cn/saml`  ← **中国区端点**
- Save。

**Client → Settings**：

- Name ID format：`persistent`（或 `email`）。
- Force POST binding：On。Sign assertions：On。Client signature required：Off。

### 3a. AWS 必需的 SAML 属性映射器

在该客户端的专用 scope 下 → **Add mapper → By configuration**：

**(i) Role** —— *Role list* 映射器：

| 字段 | 值 |
|------|-----|
| Name | `aws-role` |
| SAML attribute name | `https://aws.amazon.com/SAML/Attributes/Role` |
| NameFormat | `Basic` |

值为 `<role_arn>,<provider_arn>` 这对组合，使用 **aws-cn** 分区：

```
arn:aws-cn:iam::<CN_ACCOUNT_ID>:role/<FED_ROLE>,arn:aws-cn:iam::<CN_ACCOUNT_ID>:saml-provider/<SAML_PROVIDER>
```

**(ii) RoleSessionName** —— *User Property* 映射器：

| 字段 | 值 |
|------|-----|
| Property | `username`（或 `email`） |
| SAML attribute name | `https://aws.amazon.com/SAML/Attributes/RoleSessionName` |
| NameFormat | `Basic` |

**(iii) SessionDuration** —— *Hardcoded attribute* 映射器：

| 字段 | 值 |
|------|-----|
| SAML attribute name | `https://aws.amazon.com/SAML/Attributes/SessionDuration` |
| Attribute value | `3600`（≤ 角色的最大会话时长） |
| NameFormat | `Basic` |

> 上面的 SAML **属性名**在中国区仍是 `aws.amazon.com/...`——它们是固定的 AWS SAML claim 键。
> 只有 **ACS 端点**和 **ARN** 改为中国区的值。

### 3b. 导出 IdP 元数据

```bash
curl -s <KC_URL>/realms/aws-realm/protocol/saml/descriptor -o keycloak-idp-metadata.xml
```

## 4. AWS 中国侧 —— IAM SAML 提供商 + 联合角色

### 4a. 创建 SAML 提供商

```bash
aws iam create-saml-provider \
  --name <SAML_PROVIDER> \
  --saml-metadata-document file://keycloak-idp-metadata.xml \
  --region cn-northwest-1
# 返回：arn:aws-cn:iam::<CN_ACCOUNT_ID>:saml-provider/<SAML_PROVIDER>
```

### 4b. 创建联合角色

`trust-policy.json`（注意 `arn:aws-cn` 与**中国区** SAML 受众）：

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "Federated": "arn:aws-cn:iam::<CN_ACCOUNT_ID>:saml-provider/<SAML_PROVIDER>"
      },
      "Action": "sts:AssumeRoleWithSAML",
      "Condition": {
        "StringEquals": {
          "SAML:aud": "https://signin.amazonaws.cn/saml"
        }
      }
    }
  ]
}
```

```bash
aws iam create-role --role-name <FED_ROLE> \
  --assume-role-policy-document file://trust-policy.json \
  --max-session-duration 3600

# 附加最小权限的 QuickSight 权限（生产环境请收紧）：
aws iam put-role-policy --role-name <FED_ROLE> --policy-name quicksight-access \
  --policy-document '{
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Action": ["quicksight:CreateUser", "quicksight:RegisterUser"],
      "Resource": "*"
    }]
  }'
```

这里的角色 ARN **必须**与 Keycloak Role 属性（第 3a-i 步）中的角色 ARN 一致。

> 依赖此路径前请**核实 QuickSight 在 cn-northwest-1 的可用性**；QuickSight（Amazon Quick）
> 在中国的服务覆盖可能与商业区不同。

## 5. 测试 SAML 流程

1. IdP 发起：
   `<KC_URL>/realms/aws-realm/protocol/saml/clients/urn:amazon:webservices` → 登录 →
   应以联合角色身份登入**中国区** AWS 控制台（`console.amazonaws.cn`）。
2. 提示 *"invalid SAML response"* → 按顺序检查：Role 属性对为 `role,provider` 且使用 `arn:aws-cn`；元数据为最新；断言已签名；`SAML:aud` = `https://signin.amazonaws.cn/saml`。

## 6. 可选：自动化 Realm 配置

```bash
kubectl -n keycloak exec -it deploy/keycloak -- \
  /opt/keycloak/bin/kcadm.sh config credentials \
    --server http://localhost:8080 --realm master \
    --user admin --password "$KEYCLOAK_ADMIN_PASSWORD"

kubectl -n keycloak exec -it deploy/keycloak -- \
  /opt/keycloak/bin/kcadm.sh get realms/aws-realm/partial-export -r aws-realm \
  > aws-realm-export.json
```

用 `kcadm.sh create realms -f aws-realm-export.json`（或 `keycloak-config-cli`）重新导入，
实现客户可重复交付。

## 汇总检查清单

- [ ] Realm `aws-realm`，已开启暴力破解防护，已创建测试用户。
- [ ] OIDC 公开客户端 `amazon-quick-desktop`（PKCE S256，回环/`quick://` 回调）。
- [ ] SAML 客户端 `urn:amazon:webservices`，ACS `https://signin.amazonaws.cn/saml`。
- [ ] 三个 SAML 映射器：Role（`arn:aws-cn` 的 `role,provider` 对）、RoleSessionName、SessionDuration。
- [ ] 已创建 IAM SAML 提供商（`arn:aws-cn:iam::...:saml-provider/...`）。
- [ ] 联合角色，含 `sts:AssumeRoleWithSAML` 信任（`SAML:aud = signin.amazonaws.cn/saml`）+ 最小权限策略。
- [ ] Keycloak 中的角色 ARN 与 IAM 角色 ARN 一致。
- [ ] SAML 测试登录 `console.amazonaws.cn` 成功。
